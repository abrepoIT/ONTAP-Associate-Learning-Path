MODULE 1 - INTRODUZIONE A BLUEXP
==================================

1.1 Cos'è NetApp BlueXP?
--------------------------
NetApp BlueXP è un control plane unificato e cloud-based che fornisce una singola interfaccia di
gestione per tutti i servizi storage e dati NetApp — che siano on-premises, in un cloud privato, o
distribuiti su più cloud pubblici.

BlueXP è erogato come piattaforma Software-as-a-Service (SaaS), accessibile da qualsiasi browser su
https://bluexp.netapp.com, senza installazione locale richiesta per la maggior parte delle
funzionalità.

IL PROBLEMA CHE BLUEXP RISOLVE
Senza BlueXP, gestire un ambiente storage hybrid multicloud richiede di navigare tra più
interfacce disconnesse:
  ONTAP on-premises (AFF/FAS/ASA)           - ONTAP System Manager (per cluster)
  Cloud Volumes ONTAP (AWS/Azure/GCP)       - CLI CVO o System Manager per cluster
  Amazon FSx for NetApp ONTAP               - AWS Console + ONTAP System Manager
  Azure NetApp Files                        - Azure Portal
  Google Cloud NetApp Volumes               - Google Cloud Console
  StorageGRID (object storage)              - Interfaccia di gestione StorageGRID

BlueXP consolida tutto questo in un'unica esperienza operativa — stessa interfaccia, stessi
workflow, stesse policy — indipendentemente da dove risiedono i dati.

FILOSOFIA DI DESIGN DI BLUEXP
BlueXP è costruito attorno a tre principi fondamentali:

  DISCOVER            MANAGE              PROTECT
  Auto-scoperta       Canvas unificato    Backup & Recovery
  di tutti gli ambi-  per tutti gli ambi- Disaster Recovery
  enti NetApp         enti; Provisioning; Ransomware Protection
  On-prem + multi-    Monitoraggio;       Replication;
  cloud               Automazione         Integrazione SnapCenter

  GOVERN              OPTIMIZE            MOVE
  Classification      Digital Advisor     Copy and Sync
  Digital Wallet      Sustainability      Tiering (FabricPool)
  Compliance          Cost efficiency     Volume Caching

AMBIENTI SUPPORTATI DA BLUEXP

  ONTAP Select                       - VM su hardware commodity       - On-prem/private cloud

2.1 BlueXP Canvas e BlueXP Connector
---------------------------------------

BLUEXP CANVAS
Il Canvas è la home page visuale di BlueXP — una mappa interattiva che visualizza tutti gli ambienti
storage connessi come icone raggruppate per posizione (on-premises, AWS, Azure, GCP).

Elementi del Canvas:
  Working Environment  - Un'icona che rappresenta un sistema storage specifico (cluster ONTAP,
                         CVO, FSx, ANF, ecc.)
  Service connector    - Una linea tracciata tra un working environment e un servizio BlueXP
                         per attivarlo
  Canvas tab           - Filtra per: My estate (tutto), On-premises, AWS, Azure, GCP
  Health indicator     - Badge codificato per colore su ogni icona (verde=sano, giallo=warning,
                         rosso=critico)
  Replication line     - Rappresentazione visuale delle relazioni SnapMirror attive tra working
                         environment

Come usare il Canvas:
  1. Clicca un'icona working environment -> apre il pannello di gestione per quel sistema
  2. Trascina un'icona servizio su un working environment -> attiva quel servizio
  3. Clicca la linea tra due environment -> mostra i dettagli della relazione attiva

Tipi di working environment nel Canvas:
  Icona cluster ONTAP    - Cluster AFF/FAS/ASA on-premises
  Icona CVO single node  - Cloud Volumes ONTAP a nodo singolo
  Icona CVO HA pair      - Cloud Volumes ONTAP in alta disponibilità
  Icona FSx              - Amazon FSx for NetApp ONTAP
  Icona ANF              - Account Azure NetApp Files
  Icona GCNV             - Google Cloud NetApp Volumes
  Icona StorageGRID      - Object storage on-premises
  Icona S3/Blob/GCS      - Cloud object storage (per target di backup)

BLUEXP CONNECTOR
Il BlueXP Connector è una macchina virtuale Linux leggera deployata nell'ambiente del cliente (cloud
VPC o rete on-premises) che agisce da proxy tra la piattaforma SaaS BlueXP e l'infrastruttura
storage locale.

Perché è necessario un Connector:
  - BlueXP SaaS gira nel cloud di NetApp — non può raggiungere direttamente i cluster ONTAP
    on-premises (che sono dietro firewall, non pubblicamente accessibili)
  - Il Connector colma questo gap: si trova nella rete del cliente e trasmette i comandi di
    gestione tra BlueXP e ONTAP

Architettura del Connector:
  Internet
       |
  BlueXP SaaS (ospitato da NetApp)
       | HTTPS outbound (porta 443) — il Connector avvia la connessione
       | Nessuna regola firewall inbound richiesta
  BlueXP Connector (VM Linux — deployata dal cliente)
       | HTTPS (porta 443) verso la LIF di gestione ONTAP
       | Solo rete interna — nessuna esposizione pubblica
  Cluster ONTAP / CVO / FSx

Caratteristiche chiave del Connector:
  - Comunica solo in outbound verso BlueXP SaaS (nessuna porta inbound richiesta)
  - Può essere deployato in AWS (EC2), Azure (VM), GCP (GCE), o on-premises (VM Linux)
  - Un singolo Connector può gestire più working environment in diversi cloud provider
  - Più Connector possono essere deployati per ridondanza o separazione geografica
  - Requisiti Connector: Linux, Docker, accesso HTTPS outbound

Opzioni di deployment del Connector:
  AWS (EC2)          - Gestisce: CVO in AWS, FSx, on-prem via VPN
  Azure (VM)         - Gestisce: CVO in Azure, ANF, on-prem via VPN
  GCP (GCE)          - Gestisce: CVO in GCP, GCNV, on-prem via VPN
  On-premises (Linux) - Gestisce: ONTAP on-prem, StorageGRID (nessuna dipendenza cloud)

Quando non è necessario il Connector:
  - Amazon FSx for NetApp ONTAP (scoperta base)
  - Azure NetApp Files (scoperta base)
  - BlueXP Digital Advisor (usa dati AutoSupport già nel cloud NetApp)


2.2 BlueXP Identity and Access Management (IAM)
-------------------------------------------------
BlueXP fornisce un modello IAM multi-livello completo che consente alle organizzazioni di
controllare chi può accedere a quali risorse ed eseguire quali azioni.

GERARCHIA IAM BLUEXP
  NetApp Support Site (NSS) Account
       |
  BlueXP Organization (contenitore di primo livello)
       |
       +-- Folder (raggruppamento opzionale)
       |     +-- Project (contenitore risorse)
       |         +-- Working Environment (sistema storage)
       |
       +-- Project (contenitore risorse)
           +-- Working Environment (sistema storage)

Livelli IAM:
  Organization     - Contenitore di primo livello per tutte le risorse BlueXP dell'azienda
  Folder           - Raggruppamento opzionale di Project (es. per regione, dipartimento)
  Project          - Contenitore logico per working environment; accesso con scope al Project
  Working Env.     - Sistema storage individuale (cluster ONTAP, CVO, FSx, ecc.)

RUOLI BLUEXP
  Organization Admin         - Accesso completo a tutte le risorse, utenti, billing, connector
  Folder or Project Admin    - Accesso completo nello scope assegnato
  Workspace Viewer           - Accesso read-only ai working environment nel progetto
  Ransomware Protection Admin - Gestisce policy di protezione ransomware e risposte
  Ransomware Protection Viewer - Dashboard ransomware read-only
  SnapCenter Admin           - Gestisce integrazione SnapCenter in BlueXP
  Classification Viewer      - Visualizza risultati BlueXP Classification (no config)

Metodi di autenticazione:
  NetApp cloud login  - Account BlueXP nativo (email + password)
  Federated SSO       - Provider di identità aziendale via SAML 2.0 (Azure AD, Okta, ADFS)
  Google / GitHub     - Login social per utenti individuali

Best practice: usa SSO federato così l'accesso BlueXP è gestito dall'IdP aziendale — la
disabilitazione degli account AD revoca automaticamente l'accesso BlueXP.

SERVICE ACCOUNT E API TOKEN
Per l'automazione (pipeline CI/CD, script, Terraform):
  - Crea BlueXP Service Account (identità non umane)
  - Genera token API con scope a ruoli e progetti specifici
  - I token possono essere ruotati senza impattare gli utenti umani
  - L'audit trail mostra quale service account ha eseguito quale azione


2.3 Gestione Storage BlueXP
-----------------------------

SCOPERTA DELLO STORAGE ESISTENTE
BlueXP può scoprire automaticamente i cluster ONTAP on-premises:

Metodi di scoperta:
  1. Scoperta manuale: inserisci l'IP di gestione cluster, username e password — BlueXP si
     connette e importa tutti i dettagli del cluster
  2. Auto-scoperta via Active IQ: se il cluster è registrato con il supporto NetApp
     (AutoSupport abilitato), BlueXP può scoprirlo automaticamente dall'associazione NSS
  3. Subnet scan: il BlueXP Connector può scansionare una subnet alla ricerca di IP di
     gestione ONTAP

Cosa BlueXP scopre per ogni cluster:
  - Tutti i nodi e gli HA pair
  - Tutti gli aggregate (capacità, efficienza, salute)
  - Tutti gli SVM (protocolli abilitati, LIF, volumi)
  - Tutti i volumi (dimensione, utilizzo, stato protezione, efficienza)
  - Tutte le relazioni SnapMirror
  - Versioni firmware e software ONTAP
  - Stato e date di scadenza delle licenze

DEPLOY DI NUOVO STORAGE CLOUD

Deploy di Cloud Volumes ONTAP (CVO):
  Step-by-step dal Canvas BlueXP:
    1. Clicca Add Working Environment -> Seleziona cloud provider (AWS/Azure/GCP)
    2. Seleziona tipo deploy: Single Node o HA Pair
    3. Scegli regione, AZ, VPC/VNet
    4. Seleziona tipo istanza/VM (determina performance compute e rete)
    5. Configura storage: tipo disco (gp3/io1 per AWS, Premium SSD per Azure), dimensione
    6. Imposta credenziali ONTAP e nome SVM
    7. Revisiona e Deploy — BlueXP fa provisioning di tutte le risorse cloud automaticamente:
       istanze cloud, networking, storage volumes, configurazione ONTAP

  Tempo di deploy: circa 20-30 minuti end-to-end.

Gestione volumi tra ambienti:
Dal Canvas BlueXP, cliccando qualsiasi working environment si apre un'interfaccia di
gestione consistente:
  Overview     - Riepilogo capacità, snapshot performance, eventi recenti
  Volumes      - Lista tutti i volumi con dimensione, utilizzo, protocollo, protezione
  Aggregates   - Lista aggregate con capacità e stato RAID
  Replication  - Relazioni SnapMirror attive (sorgente e destinazione)
  Protection   - Policy Snapshot, stato backup, stato ARP
  Settings     - Configurazione SVM, protocolli, rete, efficienza

Operazioni su volumi disponibili da BlueXP (per tutti gli ambienti):
  - Crea, elimina, ridimensiona volumi
  - Abilita/modifica Snapshot policy
  - Abilita deduplication, compressione, policy di tiering
  - Imposta policy QoS
  - Crea e gestisce relazioni SnapMirror
  - Abilita cifratura NVE
  - Abilita ARP (Autonomous Ransomware Protection)

3.1 BlueXP Digital Advisor
----------------------------
BlueXP Digital Advisor (powered by Active IQ) è il layer di intelligenza AI-driven di BlueXP.
Analizza continuamente la telemetria AutoSupport da tutti i sistemi NetApp registrati e fornisce
raccomandazioni proattive e actionable per prevenire guasti, ottimizzare le performance e ridurre i
rischi.

Come funziona BlueXP Digital Advisor:
  Cluster ONTAP
       | messaggi AutoSupport (performance giornaliero + config settimanale)
  Infrastruttura AutoSupport NetApp (cloud-hosted)
       | Ingestione e preprocessing telemetria
  Active IQ AI/ML Engine
    +-- Elabora dati da 30.000+ sistemi clienti NetApp globalmente
    +-- Addestra modelli predittivi (guasti drive, capacità, performance)
    +-- Identifica deviazioni dalla configurazione dalle best practice
    +-- Genera raccomandazioni classificate per rischio
       |
  BlueXP Digital Advisor Dashboard
    +-- Presentato al cliente nell'UI BlueXP con lista azioni prioritizzate

SEZIONI DEL DASHBOARD DIGITAL ADVISOR

Wellness Score:
  - Un punteggio 0-100 per ogni sistema ONTAP che riflette la salute complessiva
  - Suddiviso in categorie di rischio:

  Availability  - Guasti disco, RAID degradato, HA non configurato, riavvii nodi
  Performance   - Aggregate sovraccaricato, alta latenza, QoS mal configurato
  Capacity      - Aggregate >85%, volume thin-provisioned vicino al limite
  Security      - Volumi non cifrati, account admin senza MFA, password default
  Configuration - Deviazione da best practice NetApp, bug noti attivi
  Efficiency    - Deduplication disabilitata, FabricPool non configurato

Watchlist:
  - Raggruppa più sistemi ONTAP in una watchlist per il monitoraggio
  - Crea watchlist per: cliente, sito, ambiente (produzione vs dev), area geografica
  - Visualizza punteggi wellness aggregati e rischi su tutti i sistemi

Upgrade Advisor:
  - Raccomanda il percorso di upgrade ONTAP ottimale dalla versione corrente
  - Valuta i rischi di upgrade: bug noti corretti nella versione target vs nuovi bug
  - Mostra quali bug nella versione corrente stanno impattando attivamente il sistema
  - Genera un piano di upgrade scaricabile con pre-check e post-check

Inventory:
  - Inventario completo di tutti i sistemi NetApp registrati: numero seriale, modello,
    versione ONTAP, stato e scadenza contratto di supporto, date EOL/EOSL, licenze

Performance:
  - Trend di performance storici (fino a 12 mesi) per cluster, nodo e volume
  - Identifica anomalie di performance (picchi, alta latenza sostenuta, saturazione IOPS)

Capacity:
  - Previsione "days-to-full" per aggregate e volume (in base al tasso di crescita)
  - Identifica opportunità di recupero spazio (snapshot obsoleti, volumi sovra-provisionati)


3.2 BlueXP Operational Resiliency
-----------------------------------
BlueXP Operational Resiliency è un servizio proattivo che identifica i rischi di firmware e software
sui sistemi ONTAP e genera piani di remediation automatizzati.

Cosa monitora Operational Resiliency:
  Firmware disco        - Drive con firmware con bug noti (perdita dati, performance)
  Firmware shelf        - Controller shelf con firmware obsoleto
  Firmware SP/BMC       - Rischi firmware Service Processor / Baseboard Management Controller
  Bug software ONTAP    - Bug ID attivi che impattano la versione ONTAP installata
  Firmware NIC/HBA      - Adapter rete e storage con problemi noti
  Rischi configurazione - Impostazioni che creano rischi di stabilità o esposizione dati

Remediation automatizzata dei rischi:
Per ogni rischio identificato, Operational Resiliency fornisce:
  1. Descrizione del rischio: qual è il problema e il suo potenziale impatto
  2. Sistemi interessati: quali cluster e nodi sono impattati
  3. Piano di remediation: passi specifici per risolvere (aggiorna firmware, cambia config)
  4. Opzione di esecuzione automatizzata: per alcuni rischi, BlueXP può eseguire la
     remediation direttamente via ONTAP REST API — cliccando Fix viene applicata la modifica
  5. Valutazione impatto: se la remediation richiede una finestra di manutenzione pianificata

Esempi di remediation automatizzate:
  - Aggiorna firmware disco tramite il meccanismo di aggiornamento non distruttivo di ONTAP
  - Applica modifiche di configurazione ONTAP per allinearsi alle best practice
  - Abilita AutoSupport sui sistemi dove era disabilitato

  Monitoraggio progresso       - Tracking progresso real-time per nodo durante l'upgrade

Workflow Software Update:
  BlueXP Software Updates Dashboard
       |
       +-- Visualizza: tutti i cluster e le loro versioni ONTAP correnti
       +-- Identifica: cluster sotto la versione raccomandata
       +-- Seleziona: versione ONTAP target (da raccomandazione Digital Advisor)
       |

    +-- HA pair operativo?                    SI'/NO
    +-- Relazioni SnapMirror sane?            SI'/NO
    +-- Nessun volume move in corso?          SI'/NO

    +-- Nessun evento EMS critico?            SI'/NO

       |

    +-- Node 2: takeover -> upgrade -> giveback
       |

    +-- Tutte le LIF sulle home port?
    +-- Versione cluster confermata?

4.1 BlueXP Backup and Recovery
--------------------------------
BlueXP Backup and Recovery è un backup-as-a-service gestito che protegge i volumi ONTAP inviando
copie di backup policy-driven verso cloud object storage — senza richiedere un sistema ONTAP
secondario come target di backup.

Sorgenti e destinazioni supportate:

Sorgenti di backup:
  - Cluster ONTAP on-premises (AFF, FAS, ASA)
  - Cloud Volumes ONTAP (AWS, Azure, GCP)
  - Amazon FSx for NetApp ONTAP
  - Azure NetApp Files / Google Cloud NetApp Volumes
  - Persistent volume Kubernetes (via Astra Trident)

Destinazioni di backup:
  AWS         - Amazon S3        - Tier disponibili: Standard -> S3-IA -> Glacier Instant
                                   -> Glacier Deep Archive
  Azure       - Azure Blob       - Tier: Hot -> Cool -> Archive
  GCP         - Google GCS       - Tier: Standard -> Nearline -> Coldline -> Archive
  On-premises - NetApp StorageGRID - Standard -> Archive
  On-premises - ONTAP S3          - Standard

COME FUNZIONA BLUEXP BACKUP

Backup iniziale (baseline completo):
  1. BlueXP legge tutti i blocchi nel volume ONTAP selezionato
  2. I dati vengono deduplicati e compressi da BlueXP (prima del trasferimento cloud)
  3. I dati vengono cifrati (AES-256) prima di lasciare l'ambiente ONTAP
  4. Il dataset completo viene caricato nell'object storage cloud come oggetto backup

Backup incrementali (dopo la baseline):
  1. BlueXP identifica i blocchi modificati dall'ultimo backup (tramite change tracking ONTAP)
  2. Solo i blocchi modificati vengono trasferiti al cloud (modello incremental-forever)
  3. Ogni backup nel cloud è un punto di restore completo e indipendente (nessuna dipendenza)
  4. BlueXP mantiene un catalogo backup per ricerca file veloce

Efficienza incremental-forever:
  - Dopo la baseline iniziale, la crescita del cloud rispecchia il tasso di modifica dati
  - Nessun backup completo settimanale/mensile necessario (elimina la backup window)
  - Il restore da qualsiasi punto usa il catalogo per ricostruire lo stato del volume

BLUEXP BACKUP POLICY — ESEMPIO
  Policy: "production-backup"
  Orario:    mantieni 24 copie
  Giornaliero: mantieni 30 copie -> sposta in tier IA dopo 30 giorni
  Settimanale: mantieni 13 copie
  Mensile:   mantieni 12 copie -> sposta in tier Archive dopo 180 giorni
  Annuale:   mantieni 7 copie
  DataLock:  Abilitato (Governance o Compliance mode)

DATALOCK E PROTEZIONE RANSOMWARE

DataLock è la funzionalità di immutabilità a livello cloud di BlueXP:
  Governance mode  - Immutabile con override; chi può cancellare prima della scadenza:
                     Admin privilegiato con permesso speciale
  Compliance mode  - Strettamente immutabile; chi può cancellare: nessuno (nemmeno root)

Implementazione DataLock:
  - AWS: integra con S3 Object Lock (WORM storage a livello S3)
  - Azure: integra con Azure Blob Immutable Storage
  - GCP: integra con GCS Object Hold

Scansione ransomware al restore:
Prima di ripristinare da un backup cloud, BlueXP esegue una scansione ransomware inline:
  - Scansiona per firme ransomware note nei dati di backup
  - Genera alert se vengono rilevati pattern cifrati/sospetti
  - Impedisce il restore di un backup preso dopo l'infezione ransomware

Opzioni di restore:
  Volume restore         - Ripristina intero volume in posizione originale o nuova - Min-Ore
  File/folder browse     - Sfoglia catalogo e ripristina file specifici             - Minuti
  Quick restore          - Accesso volume istantaneo via FlexClone dal backup cloud  - Secondi
  Cross-region restore   - Ripristina in una diversa regione cloud                  - Min-Ore
  Cross-account restore  - Ripristina in un diverso account cloud                   - Min-Ore


4.2 Integrazione SnapCenter con BlueXP Backup and Recovery
------------------------------------------------------------
SnapCenter e BlueXP Backup and Recovery sono complementari — SnapCenter fornisce backup application-
aware e policy-based a livello applicativo, mentre BlueXP estende questi al cloud per la protezione
offsite.

Architettura di integrazione:
  Host applicativo (SQL Server, Oracle, VMware)
       | Snapshot application-consistent (VSS/RMAN quiesce)
  SnapCenter Server
       | Orchestra backup + registra nel catalogo
  Snapshot ONTAP (application-consistent)
       |
       +-- Locale: conservato secondo policy SnapCenter (orario/giornaliero/settimanale)
       |
       +-- BlueXP Backup and Recovery
               | Scopre snapshot gestiti da SnapCenter
           Cloud Object Storage (S3 / Blob / GCS)
               +-- Retention a lungo termine con protezione DataLock

Cosa abilita l'integrazione:
  - Catalogo unificato: il catalogo backup BlueXP include backup applicativi gestiti da
    SnapCenter
  - Estensione cloud: gli snapshot applicativi SnapCenter possono essere copiati
    automaticamente nel cloud per la retention offsite
  - Unica interfaccia di restore: gli admin applicativi possono ripristinare da snapshot
    SnapCenter locali e copie cloud dalla stessa interfaccia BlueXP
  - Allineamento policy: le policy RPO SnapCenter definiscono la retention locale; le policy
    BlueXP definiscono la retention cloud

Registrazione SnapCenter in BlueXP:
  BlueXP -> Backup and Recovery -> Applications -> Register SnapCenter Server
    +-- Inserisci FQDN/IP server SnapCenter e credenziali
    +-- BlueXP scopre tutte le risorse gestite da SnapCenter:
        +-- Istanze e database SQL Server
        +-- Database Oracle
        +-- Exchange DAG
        +-- Sistemi SAP HANA


4.3 Monitoraggio Data Protection
----------------------------------
BlueXP fornisce un dashboard di monitoraggio data protection unificato che aggrega lo stato di
protezione su tutti gli ambienti connessi.

Dashboard monitoraggio protezione (BlueXP -> Backup and Recovery -> Dashboard):
  Backup status         - % volumi con policy backup attive; volumi senza backup
  Jobs                  - Job di backup recenti (successo, fallimento, in corso) con durata
  Alerts                - Backup falliti, policy scadute, warning soglie storage
  SnapMirror health     - Stato relazioni su tutti i cluster connessi
  Ransomware incident   - Rilevamenti ARP e alert protezione ransomware BlueXP
  Protected data volume - TB totali sotto protezione backup
  Cost summary          - Costo storage backup cloud mensile stimato

Monitoraggio Job:
Per ogni job di backup, BlueXP registra: tipo job, working environment sorgente, orari
inizio/fine/durata, dati trasferiti (MB/GB), stato, motivo del fallimento e guida al
troubleshooting.

Report conformità protezione:
  - Quali volumi sono protetti vs non protetti
  - Quali volumi hanno violazioni RPO (ultimo backup più vecchio di quanto la policy richiede)
  - Sistemi storage senza servizio backup abilitato
  - Stato conformità DataLock


4.4 BlueXP Disaster Recovery
------------------------------
BlueXP Disaster Recovery è un servizio di orchestrazione DR gestito che automatizza il failover e il
failback dei workload VMware usando la replica ONTAP SnapMirror — eliminando la necessità di
operazioni manuali SnapMirror CLI durante un disastro.

Architettura BlueXP DR:
  Sito A (Protetto / Primario)                   Sito B (Recovery)
  vCenter A / Cluster ONTAP A                    vCenter A' o vCenter B / Cluster ONTAP B
  VM -> volumi datastore ONTAP    --SnapMirror--> Copie DR

  BlueXP DR Service (SaaS):
    +-- Scopre VM VMware e il loro storage ONTAP
    +-- Mappa VM -> datastore -> volume ONTAP -> relazione SnapMirror
    +-- Mantiene il DR Plan (ordine boot VM, mappature rete, mappature risorse)
    +-- Orchestra failover/failback via REST API verso vCenter + ONTAP

Componenti di un DR Plan:
  VM protette         - Quali VM sono incluse in questo piano DR
  Ordine di boot      - Sequenza di accensione VM al sito di recovery
  Ritardo boot        - Tempo di attesa tra operazioni di accensione VM
  Mappature rete      - Rete sorgente -> rete recovery (rimappatura IP se necessario)
  Mappature risorse   - Compute sorgente (host/cluster ESXi) -> compute recovery
  Mappature datastore - Volumi ONTAP sorgente -> volumi ONTAP recovery

Operazioni DR:

Test failover (non distruttivo):
  1. BlueXP crea FlexClone dei volumi di destinazione SnapMirror (nessun impatto sulla
     replica)
  2. Le VM vengono accese nel vCenter di recovery usando i datastore clone
  3. Viene eseguito il testing — l'isolamento rete evita conflitti IP
  4. Dopo il test: i clone vengono eliminati, SnapMirror di produzione continua

Failover pianificato (manutenzione schedulata):
  1. BlueXP mette in quiesce le VM sorgente (shutdown controllato)
  2. Aggiornamento SnapMirror finale garantisce RPO = 0
  3. SnapMirror viene interrotto sulla destinazione
  4. Le VM vengono registrate e accese nel vCenter di recovery
  5. Mappature di rete applicate

Failover non pianificato (disastro):
  1. Il sito sorgente diventa irraggiungibile
  2. L'amministratore attiva il failover da BlueXP DR
  3. BlueXP interrompe SnapMirror sulla destinazione
  4. Le VM vengono accese al sito di recovery secondo l'ordine boot del piano DR
  5. RTO misurato in minuti

Failback (ritorno al primario):
  1. Il sito primario viene ripristinato e la connettività verificata
  2. BlueXP attiva la replica inversa (sito recovery -> sito primario)
  3. Le modifiche durante il periodo DR vengono sincronizzate
  4. Failback pianificato: quiesce VM recovery -> sync -> accendi primario

BlueXP DR vs VMware SRM:
  Infrastruttura richiesta: BlueXP = SaaS (nessuna VM aggiuntiva) / SRM = Server + plugin SRA
  Integrazione ONTAP:       BlueXP = Nativa (no plugin SRA) / SRM = Richiede OTV SRA adapter
  Gestione DR plan:         BlueXP = UI BlueXP / SRM = Interfaccia vCenter SRM
  Test non distruttivo:     BlueXP = Sì (FlexClone) / SRM = Sì (FlexClone via SRA)
  Licenza:                  BlueXP = Inclusa in BlueXP/ONTAP One / SRM = Licenza VMware
  Ideale per:               BlueXP = DR VMware NetApp-centrico / SRM = Deploy SRM esistenti

7.1 BlueXP Copy and Sync
--------------------------
BlueXP Copy and Sync (ex Cloud Sync) è un servizio di migrazione e sincronizzazione dati che sposta
o sincronizza dati tra qualsiasi due endpoint storage — on-premises o cloud — con supporto per
storage NetApp e non-NetApp.

Combinazioni sorgente-destinazione supportate:
Sorgenti (e destinazioni): share NFS, share SMB/CIFS, Amazon S3, Azure Blob Storage, Google Cloud
Storage, IBM Cloud Object Storage, ONTAP S3, StorageGRID, server SFTP, Box, Dropbox.

Casi d'uso chiave:
  Migrazione cloud        - Sposta dati NFS da NAS on-prem verso AWS S3
  Replica multi-cloud     - Sincronizza bucket S3 verso Azure Blob per ridondanza
  Consolidamento NAS      - Migra share SMB da più file server a ONTAP NAS centralizzato
  Sincronizzazione ibrida - Mantieni copie on-prem e cloud sincronizzate per accesso ibrido
  Preparazione DR         - Sincronizza dati critici verso cloud come parte della strategia DR
  Migrazione vendor       - Migra da NAS competitor a NetApp ONTAP

Come funziona Copy and Sync:
  BlueXP Copy and Sync Service (SaaS)
       |
  Data Broker (agente leggero deployato nell'ambiente del cliente)
       |
       +-- Legge dalla sorgente (NFS, SMB, S3, ecc.)
       +-- Trasferisce dati (percorso diretto — non passa per il cloud SaaS BlueXP)
       +-- Scrive sulla destinazione (NFS, SMB, S3, Blob, ecc.)

Data Broker: deployato in AWS (Lambda o EC2), Azure (VM), GCP (GCE), o on-premises (Docker). I dati
fluiscono direttamente tra sorgente e destinazione attraverso il broker — non attraverso il cloud
NetApp. Più broker possono formare un broker group per trasferimenti paralleli ad alto throughput.

Funzionalità delle relazioni di sync:
  One-time copy      - Migra dati una volta (nessuna sync continua)
  Continuous sync    - Sincronizzazione continua; rileva e replica modifiche
  Sync schedule      - Trigger orario, giornaliero, settimanale
  File filtering     - Includi/escludi per estensione, dimensione, data modifica, regex
  Cifratura transito - Cifratura TLS per tutti i dati in transito
  Cifratura a riposo - Cifratura per destinazioni S3/Blob/GCS
  ACL preservation   - Preserva permessi NTFS tra share SMB
  Metadata pres.     - Preserva timestamp, ownership e attributi
  Continuous mode    - Sync quasi real-time usando API change notification (S3 events, ecc.)

Monitoraggio Copy and Sync:
  - Dashboard relazioni: stato, ultimo tempo sync, file copiati, errori
  - Storico per relazione: log dettagliato di ogni esecuzione sync
  - Error reporting: file che non sono stati copiati con codici di errore specifici
  - Notifiche: alert email per sync fallite o completate


7.2 BlueXP Tiering
--------------------
BlueXP Tiering è l'interfaccia di gestione per NetApp FabricPool — la tecnologia di tiering
automatico dei dati di ONTAP che sposta i dati freddi (acceduti raramente) dallo storage flash
costoso verso cloud object storage a basso costo.

FabricPool crea un sistema storage a due tier:
  Performance tier (locale): NVMe/SSD su ONTAP — serve dati caldi con latenza sub-millisecondo
  Capacity tier (cloud):     Object storage S3/Blob/GCS — memorizza dati freddi a 1/10 del costo

Lo spostamento dati è trasparente per le applicazioni — le applicazioni leggono e scrivono
normalmente; ONTAP gestisce il tiering in background.

Capacità di BlueXP Tiering:
  Scoperta opportunità   - Analizza tutti i cluster ONTAP on-prem e identifica volumi con dati
                           freddi
  Stima risparmi         - Calcola risparmio costi proiettato se il tiering venisse abilitato
  Configurazione         - Imposta connessioni object store FabricPool e policy da BlueXP
  Monitoraggio           - Traccia dati tiered, tassi tiering e tassi retrieval
  Gestione policy        - Regola periodi di raffreddamento e policy di tiering per volume

BlueXP Tiering Dashboard (BlueXP -> Mobility -> Tiering -> Dashboard):
  +-- Totale dati freddi on-premises (eligibili per tiering)
  +-- Risparmio mensile stimato se tutti i dati freddi vengono tiered
  +-- Dati attualmente tiered (GB/TB) e tasso tiering (GB/giorno)
  +-- Tasso retrieval (dati freddi richiamati sul flash)
  +-- Dettaglio per cluster: nome, volume dati freddi, tiering abilitato?, risparmi

Destinazioni object store supportate:
  AWS         - Amazon S3   - Classi: Standard, Standard-IA, One Zone-IA, Glacier
  Azure       - Azure Blob  - Tier: Hot, Cool, Archive
  GCP         - GCS         - Classi: Standard, Nearline, Coldline, Archive
  On-premises - StorageGRID - Standard
  On-premises - ONTAP S3    - Standard

Policy di tiering:
  Snapshot-only - Tiera solo dati freddi negli snapshot non nel filesystem attivo
                  Ideale per: conservativo, rischio performance minimo
  Auto          - Tiera dati freddi in snapshot + dati attivi del filesystem
                  Ideale per: workload generali (più comune)
  All           - Tutti i dati immediatamente al momento della scrittura
                  Ideale per: workload di archivio/write-once
  None          - Nessun tiering — tutti i dati restano sul flash
                  Ideale per: workload latency-sensitive Tier-1

Il periodo di raffreddamento (default: 31 giorni) controlla quando i dati sono considerati freddi.
Configurabile da 2 a 183 giorni per volume. Digital Advisor suggerisce il periodo ottimale in base
ai pattern di accesso reali.


7.3 BlueXP Volume Caching
---------------------------
BlueXP Volume Caching (basato sulla tecnologia ONTAP FlexCache) crea volumi read-cache in posizioni
remote che memorizzano un sottoinsieme di dati frequentemente acceduti da un volume ONTAP di origine
centrale — riducendo la latenza e il consumo di banda WAN per utenti geograficamente distribuiti.

Il problema che risolve Volume Caching:

SENZA volume caching:
  ONTAP Centrale (HQ)               Ufficio Remoto / Cloud
  Volume Sorgente (10 TB)           Gli utenti accedono ai file
  /engineering/designs/  <---WAN--- Ogni lettura attraversa la WAN
                          50ms RTT  Alta latenza per tutte le operazioni

CON volume caching:
  ONTAP Centrale (HQ)               Sito Remoto / Cloud
  Volume Origine (10 TB)            Volume FlexCache (2 TB — subset)
  /engineering/designs/  <-WAN sync File caldi cachati localmente
                                    Gli utenti leggono dalla cache: <1ms
                                    File freddi recuperati dall'origine on demand
                                    Traffico WAN: solo cache miss e scritture

Funzionalità BlueXP Volume Caching:
  Crea volumi cache     - Deploy volumi FlexCache su ONTAP o CVO remoto da BlueXP
  Monitora efficienza   - Hit ratio, dimensione cache, riduzione traffico WAN
  Gestione policy       - Configura write delegation, regole popolamento cache
  Multi-cloud caching   - Cача dati on-premises in AWS/Azure/GCP per workload cloud
  Cache invalidation    - BlueXP gestisce la consistenza cache con il volume di origine

Casi d'uso Volume Caching:
  Collaborazione global. - Team in diverse geografie accedono a file CAD grandi localmente
  Accelerazione hybrid   - Dati on-premises cachati nella regione cloud per app cloud-native
  Media e entertainment  - Farm di rendering accedono asset media da repository centrale
  EDA                    - Team di design semiconduttori accedono librerie chip da più siti
  HPC scratch space      - Cluster compute cachano dataset di input frequentemente acceduti

Volume Caching vs BlueXP Copy and Sync:
  Modello dati:    Caching = Un'origine, cache distribuite (read-mostly)
                   Copy/Sync = Copie indipendenti
  Consistenza:     Caching = Forte (cache riflette sempre l'origine)
                   Copy/Sync = Eventual (sync schedulata)
  Scritture:       Caching = Dirette all'origine (write delegation opzionale)
                   Copy/Sync = Scritte alla destinazione indipendentemente
  Caso d'uso:      Caching = Accelera accesso distribuito read-heavy
                   Copy/Sync = Migrazione dati, sync multi-cloud
  Protocolli:      Caching = NFS, SMB
                   Copy/Sync = NFS, SMB, S3, Blob e molti altri
